# 任意ステップ: BRD をゼロから作る

> **これは何?** 本ワークショップは通常、用意済みの [docs/BRD.md](../../docs/BRD.md) を使って進めますが、
> 本来は **BRD（ビジネス要件定義）自体を作るところ**から始められるべきです。このページは、
> **ビジネスアイデアだけ**がある状態から、対話形式の **BRD 作成エージェント**で `docs/BRD.md` を
> 作る任意の手順です。
> **概念:** [0.Concepts-Primer.ja.md](0.Concepts-Primer.ja.md) §2（BRD とは）・**通常フロー:** [5.Demo-Flow.md](5.Demo-Flow.md)

---

## いつ使うか

| 状況 | 進め方 |
| --- | --- |
| BRD が既にある（既定）| 用意済みの [docs/BRD.md](../../docs/BRD.md) をそのまま使用 → HLD / Issue へ |
| **BRD がまだ無い／ゼロから作りたい** | **この任意ステップ**で `0.SDLC BRD Author` を使って作成 |

> 「要件（What/Why）が良ければ実装は安定する」— だからこそ、BRD を自分で作る体験は価値があります。

---

## 使うエージェント

- **`0.SDLC BRD Author`**（新規・対話形式）— アイデアから**インタビューして** `docs/BRD.md` を作る。
- 既存の **`1.SDLC BRD Agent`** — BRD/要求から **Epics・Features** へ分解する（こちらは分解担当）。

> 役割分担: **0 = ゼロからBRDを書く（対話）** → **1 = BRD を Epics/Features に分解** → **2 = HLD** → Issue。

---

## 手順

### 1. 入力（ビジネスアイデア）を用意
[docs/brd-brief.example.md](../../docs/brd-brief.example.md) のサンプル（契約者セルフサービス等）を
そのまま使うか、自分の業務に近いアイデアを 1〜2 段落で用意します。

### 2. BRD Author を呼び出す（Copilot Chat / Agent モード）

```
@0.SDLC BRD Author 次のビジネスアイデアから BRD を作りたいです。
不足があればインタビューしてください。
[ここにアイデア or docs/brd-brief.example.md の内容を貼り付け]
```

### 3. インタビューに答える
エージェントが不足情報を質問します（課題／対象ユーザー／成功指標／スコープ In・Out／制約／優先度）。
「お任せ」と言えば、妥当な前提を置いて進め、**前提は BRD に明記**されます。

### 4. 生成と確認
`docs/BRD.md` が作成されます。要点サマリと未確定事項を確認し、必要なら修正を依頼します。

### 5. 次へ引き継ぐ
- `@1.SDLC BRD Agent` … BRD を **Epics / Features** に分解
- `@2.SDLC HLD Agent` … BRD から **高位設計**
- その後、優先度付きバックログから **GitHub Issue** を作成（[5.Demo-Flow.md](5.Demo-Flow.md)）

---

## 完成の目安（Quality Gate）

- 各ビジネスゴールに**測定可能**な成功指標がある
- In / Out スコープが明示されている
- バックログ各項目に優先度と理由がある
- リスクに対する緩和策がある
- 計画レベルの NFR（セキュリティ/性能/使いやすさ）が記載されている
- 置いた前提が明記されている

---

## 注意

- このエージェントは **計画のみ**。コード・設計・UIは作りません。
- 出力言語は**利用者の言語に合わせます**（本ワークショップは日本語が既定）。
- 既定の [docs/BRD.md](../../docs/BRD.md) を**上書き**する場合は、元に戻せるよう git で差分を確認してください
  （別名で保存して比較してもOK）。
