# 🚀 エージェント駆動 SDLC ワークフローの流れ

## ステップごとのビジュアルガイド

### 1️⃣ リクエストフローのセットアップ
![Request Flow](../../images/0_0.request%20flow.png)
Issue からワークフローのリクエストを開始 → エージェントが処理 → コード生成

### 2️⃣ サービス接続
![Service Connection](../../images/0_1.service_connection.png)
GHE と Azure DevOps の間にパイプライン連携用のセキュアな接続を確立

### 3️⃣ Copilot コードレビュー(初回)
![Copilot Code Review](../../images/0_2_copilot%20code%20review_.png)
生成されたコードに対し Copilot が自動コードレビューを開始

### 4️⃣ ステータスチェック
![Status Checks](../../images/0_status%20checks.png)
CI/CD パイプラインが自動チェックとバリデーションを実行

### 5️⃣ 自動コードレビュー
![Automatic Review](../../images/1_automatic%20code%20review.png)
詳細なフィードバックを伴うフルの自動コードレビュー

### 6️⃣ GitHub アカウントと ADO を連携
![Link Accounts](../../images/2.link_github_account-ado.png)
GitHub と Azure DevOps の認証およびアカウント連携

### 7️⃣ ドラフト Pull Request
![Draft PR](../../images/3_draft%20pull%20request.png)
PR をドラフト状態で作成、レビュー準備完了

### 8️⃣ レビュー準備完了
![Ready for Review](../../images/4_ready_for_review.png)
PR を人間のレビュー・承認用に Ready 状態へ

### 9️⃣ Copilot で一括修正
![Fix Batch](../../images/5_fix%20batch%20with%20copilot.png)
検出された問題に対し Copilot がまとめて修正を生成

### 🔟 コミット提案
![Commit Suggestion](../../images/6_commit_suggestion.png)
Copilot がコミットメッセージと変更内容を提案

### 1️⃣1️⃣ @copilot へ委任
![Copilot Delegate](../../images/7_%40copilot_delegate.png)
開発者がタスクを委任: `@copilot apply fixes`

### 1️⃣2️⃣ 修正してコミット
![Fix and Commit](../../images/7_fix%20and%20commit.png)
Copilot が自動で修正を適用しコミット

### 1️⃣3️⃣ @copilot 実行中
![Copilot Working](../../images/8_%40copilot_working.png)
Copilot が要求された全変更を処理・最終化

---

## 🎯 主な成果

✅ Issue から PR までの自動ワークフロー  
✅ 継続的なコードレビューフィードバック  
✅ Copilot 委任による自己修復能力  
✅ Azure DevOps パイプラインとのシームレスな統合  
✅ 手動レビューサイクルの削減
