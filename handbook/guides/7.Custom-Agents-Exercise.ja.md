> **ねらい:** 用意済みのエージェントを「使う」だけでなく、**自分で 1 つ作って呼び出す**ところまでを
> 短時間で体験します。作り切れなくても OK — 環境は後日も使えるので各自で継続できます。
> **概念:** [0.Concepts-Primer.ja.md](0.Concepts-Primer.ja.md) の §4 ・ **既存エージェント:** `.github/agents/`

---

## カスタムエージェントとは（おさらい）

`.github/agents/` 配下の `.agent.md` ファイルで定義する、**役割特化のエージェント**です。
YAML フロントマター（`name` / `description` など）＋本文の指示で構成され、Copilot Chat から
`@<name>` で呼び出します。本リポジトリの既存例: `1.SDLC BRD Agent` / `2.SDLC HLD Agent` /
`5.SDLC Dev Agent` / `6.SDLC Test Agent`。

---

## 進め方（15分・タイムボックス）

1. **シナリオを選ぶ（2分）** — 下のシナリオ集から 1 つ。自分の業務に近いものでも可。
2. **エージェントを作る（7分）** — `/create-agent` を使うか、`.github/agents/` に `.agent.md` を直接作成。
3. **呼び出して試す（4分）** — Copilot Chat（Agent モード）で `@<name>` に続けて依頼。
4. **共有（2分）** — 数名に「作ったもの・狙い・気づき」を一言ずつ発表してもらう。

> 作り切れなくても問題ありません。「型」を掴むことがゴールです。

---

## 作り方 A — `/create-agent` を使う

1. Copilot Chat を **Agent モード**にする
2. 入力欄に `/create-agent` と入力し、対話に沿って役割・対象・指示を埋める
3. 生成された `.agent.md` を確認・保存

## 作り方 B — `.agent.md` を直接作成

`.github/agents/my-agent.agent.md` を作成:

```markdown
---
name: My Domain Agent
description: "Use when: <いつ使うか（1行で）>"
---

# 役割

あなたは <役割> です。<対象/前提>。

## 指示
- <やること1>
- <やること2>
- 出力は <形式（箇条書き / 表 / コードなど）> で返す

## やらないこと
- <禁止事項（例: コードは書かない / ファイルは変更しない）>
```

呼び出し:

```
@My Domain Agent <依頼内容>
```

## うまく動かすコツ

- **`name` は呼び出し名**。短く一意に（`@` 補完に出ます）。
- **`description` の "Use when:"** を具体的に書くと、Copilot が適切な場面で選びやすくなります。
- **役割を絞る**（万能より専門特化）。「やらないこと」を明記すると暴走しにくい。
- 反映されないときは VS Code を **Reload Window**。
