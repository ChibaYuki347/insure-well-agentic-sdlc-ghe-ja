# GitHub Codespaces で参加する（ローカル環境構築なしの選択肢）

> **これは何?** ローカルに Java・Node・VS Code を入れずに、ブラウザ（または VS Code）から
> クラウド開発環境でワークショップに参加するための手順です。**前提セットアップで詰まった人の
> 切り札**にもなります。
> **概念:** [0.Concepts-Primer.ja.md](0.Concepts-Primer.ja.md) ・ **ローカル派の前提:** [../setup/1.Prerequisites.ja.md](../setup/1.Prerequisites.ja.md)

---

## なぜ Codespaces を使うのか

| ローカル構築 | Codespaces |
| --- | --- |
| Java/Maven/Node/VS Code 拡張を各自インストール | **インストール不要**（コンテナに同梱）|
| 「自分の PC だけ動かない」問題が起きがち | 全員が**同一環境**で揃う |
| プロキシ・権限で詰まりやすい | ブラウザだけで起動できる |

このリポジトリには [`.devcontainer/devcontainer.json`](../../.devcontainer/devcontainer.json) が含まれており、
Codespace を作成すると **Java 17 / Maven / Node 20 / 必要な VS Code 拡張**が自動で用意されます。

> ⚠️ **社内ポリシーの確認:** 組織によっては Codespaces の利用や、forwarded port の公開範囲が
> 制限されている場合があります。**利用可否は事前に運営／管理者に確認**してください。

---

## 1. Codespace を作成する

1. GitHub 上の対象リポジトリを開く。
2. 緑の **`< > Code`** ボタン → **Codespaces** タブ → **Create codespace on main**。
3. コンテナのビルドを待つ（初回は数分）。`postCreateCommand` でフロントエンドの依存も自動インストールされます。

> VS Code デスクトップから使いたい場合は、拡張機能 **GitHub Codespaces** を入れて
> 「Open in VS Code Desktop」を選択しても同じ環境に接続できます。

---

## 2. アプリを起動する

ターミナルで:

```bash
cd src
chmod +x run.sh
./run.sh
```

- **バックエンド**: `http://localhost:8080/api`（Spring Boot）
- **フロントエンド**: `http://localhost:3000`（React）

起動すると **PORTS（ポート）** タブにポート 3000・8080 が現れます。**3000 の「ブラウザで開く」**
（地球儀アイコン）でアプリが開きます。

> 🔌 **ポート転送の仕組み:** Codespaces ではブラウザはあなたの PC 側で動くため、`localhost:8080` は
> そのままでは届きません。本リポジトリではフロントエンドが **相対パス `/api`** で呼び、
> CRA の dev サーバーがバックエンド（8080）へ**プロキシ**します（[package.json](../../src/frontend/package.json) の `proxy`）。
> そのため**ポート 3000 だけ**でアプリが完結し、8080 を公開する必要はありません。

## 3. MCP（ローカル）を Codespaces で使う

> ⚠️ **「リモート MCP 禁止」と Codespaces は両立します。** Codespaces はクラウド上の開発環境ですが、
> MCP サーバーは Codespace コンテナ内で `npx`（**stdio＝ローカル起動**）として動きます。
> これは「リモート MCP（http/sse で外部ホストに接続）」ではありません。設定方針はローカルと同じです。

- 設定は [../setup/3.MCP-Local-Setup.ja.md](../setup/3.MCP-Local-Setup.ja.md) と同じ手順。
- `.npmrc`（社内プライベートレジストリ）の設定が必要なら、Codespace 内でも同様に行います
  （URL・認証は運営から共有）。
- `.vscode/mcp.json` は運営提供の構成、または [../setup/mcp.local.json](../setup/mcp.local.json) を
  コピーして使用します。

Playwright を使う場合は、Codespace のターミナルで一度だけ:

```bash
npx playwright install chromium
```

## 4. うまくいかないとき

| 症状 | 対処 |
| --- | --- |
| 画面が真っ白／API エラー | `./run.sh` が両方起動しているか確認。PORTS に 3000・8080 があるか |
| `Invalid Host header` | devcontainer で `DANGEROUSLY_DISABLE_HOST_CHECK=true` を設定済み。再起動で解消 |
| ホットリロードが効かない | `WDS_SOCKET_PORT=0` 設定済み。ページ再読込で反映される場合あり |
| `npx` が取得できない | `.npmrc` の社内レジストリ設定・認証・プロキシを確認（ローカルと同じ）|
| MCP ツールが Chat に出ない | `Developer: Reload Window` 後に Tools を再確認 |

---

## 5. 後片付け

Codespace は使った時間で課金されます。終わったら停止/削除を:

- GitHub → 右上プロフィール → **Codespaces**、または https://github.com/codespaces
- 不要な Codespace を **Stop**（一時停止）または **Delete**（削除）。
