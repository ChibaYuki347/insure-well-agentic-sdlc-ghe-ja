# Playwright MCP ワークショップガイド — InsureWell

> このリポジトリで GitHub Copilot とともに Playwright MCP サーバーを使用する前に、本ガイドを通読してください。

このワークショップは、**InsureWell** 健康保険管理アプリ (React + Spring Boot) に対するハンズオン形式で構成されています。セットアップから始め、簡易検証を行い、その後 Dashboard と Claims UI を扱うガイド付きの MCP 演習へと進みます。

---

## 1. ワークショップのゴール

このラボの終了時点で、参加者は次のことができるようになります:

- 通常の Playwright テスト生成に対して、Playwright MCP が何を追加するかを理解する
- VS Code に Playwright MCP サーバーを構成する
- InsureWell のフルスタックアプリを起動し、設定済みのベース URL を検証する
- MCP のブラウザツールを使い、InsureWell の UI に対してナビゲーション/クリック/フォーム入力/インスペクション/挙動検証を行う
- インタラクティブな MCP 検証と、リポジトリ内の `src/frontend/tests/` にある Playwright テストスイートとを結びつける

---

## 2. Playwright MCP とは

Playwright MCP (Model Context Protocol) サーバーは、ブラウザ自動化機能を AI エージェントから呼び出せるツール群として公開します。VS Code に構成すると、Copilot がエージェント会話の中でページ遷移、要素クリック、フォーム入力、スクリーンショット、JavaScript 評価などを行えます。

これにより、次の 2 つの動作モードが可能になります:

| モード | できること |
| --- | --- |
| Generate | エージェントが Playwright のスペックファイルを書く。 |
| Generate + Execute | エージェントがテストを書き、起動中のアプリに対してインタラクティブに挙動を検証もできる。 |

## 3. アプリケーションアーキテクチャ

| レイヤー | 技術 | URL |
| --- | --- | --- |
| フロントエンド | React 18 (Create React App) | `http://localhost:3000` |
| バックエンド | Spring Boot 3 / Java | `http://localhost:8080/api` |
| データベース | インメモリ H2 (起動時にシード) | — |

フロントエンドはすべての保険契約/クレームデータをバックエンド REST API 経由で取得します。UI を動かすには両サービスが起動している必要があります。

## 5. Playwright MCP のインストールと構成

### オプション A: グローバルインストール

```bash
npm install -g @playwright/mcp
```

インストール確認:

```bash
npx @playwright/mcp --version
```

### オプション B: ワークスペース駆動の実行 (既に構成済み)

本リポジトリは `.vscode/mcp.json` 内で `npx @playwright/mcp@latest` を使うため、グローバルインストールは任意です。

### VS Code が認識しているか確認

1. Copilot Chat を開く。
2. チャット入力バーの **Tools** (プラグ) アイコンをクリック。
3. Playwright の `browser_*` ツール一覧が表示されていることを確認。
4. 見えない場合は VS Code ウィンドウをリロード (`Cmd+Shift+P` → `Developer: Reload Window`) し、Output パネルでエラーを確認。

## 6. リポジトリ設定

### MCP 設定 — `.vscode/mcp.json`

```json
{
  "servers": {
    "microsoft/playwright-mcp": {
      "type": "stdio",
      "command": "npx",
      "args": [
        "@playwright/mcp@latest"
      ],
      "env": {
        "PLAYWRIGHT_BASE_URL": "http://localhost:3000"
      }
    }
  },
  "inputs": []
}
```

### Playwright 設定 — `src/frontend/playwright.config.ts`

```ts
use: {
  baseURL: 'http://localhost:3000',
  trace: 'on-first-retry',
  screenshot: 'only-on-failure'
}
```

アプリのホストやポートが変わった場合は、両方のファイルを更新してください。

## 7. 利用可能なツール名

MCP サーバーがアクティブな状態で、以下のブラウザツールが利用できます。

### ナビゲーション

- `browser_navigate` — URL に遷移
- `browser_navigate_back` — 履歴を戻る
- `browser_resize` — ブラウザウィンドウをリサイズ
- `browser_close` — 現在のページを閉じる

### 操作

- `browser_click` — 要素をクリック
- `browser_fill_form` — 複数のフォームフィールドを一括入力
- `browser_type` — 編集可能要素にテキストを入力
- `browser_select_option` — ドロップダウンからオプションを選択
- `browser_handle_dialog` — ブラウザの alert/confirm/prompt を処理
- `browser_hover` — 要素にホバー
- `browser_press_key` — キーを押下
- `browser_file_upload` — 1 つ以上のファイルをアップロード

### インスペクション

- `browser_snapshot` — 現在ページのアクセシビリティ型スナップショット
- `browser_take_screenshot` — 現在ページのスクリーンショット
- `browser_evaluate` — ページ上で JavaScript を評価
- `browser_console_messages` — ブラウザコンソール出力を返す
- `browser_network_requests` — ページロード以降のネットワークリクエストを返す

### コンテキスト

- `browser_tabs` — タブの一覧表示/作成/クローズ/選択

## 8. 本アプリのセレクタ規約

常に `data-testid` 属性を使用してください。CSS クラスや位置ベースのセレクタは使用しないこと。

**正しい例:**

```text
[data-testid='add-policy-btn']
data-testid=new-claim-btn
```

```text
.btn-primary
button:nth-child(2)
```

## 10. ワークショップの流れ概要

スムーズなハンズオンのため、以下の順序を推奨します:

1. アプリを起動し、バックエンド (8080) とフロントエンド (3000) の動作を確認。
2. ナビゲーション検証のため、簡易プロンプトを 2 つ実行。
3. Dashboard でクリック・モーダルのワークフローを試す。
4. フォーム入力で契約を追加してみる。
5. Claims ページへの遷移を試す。
6. 新規クレームの提出を試す。
7. ドロップダウンでのフィルタとステータス更新を試す。
8. フォームのバリデーションエラー状態を試す。
9. スクリーンショットとアクセシビリティスナップショットを試す。
10. モバイル幅へのビューポート変更を試す。

## 14. トラブルシューティング

### MCP サーバーが Copilot に表示されない

1. `.vscode/mcp.json` の構文を確認 (JSON として妥当か、インデントが正しいか)。
2. VS Code ウィンドウをリロード (`Cmd+Shift+P` → `Developer: Reload Window`)。
3. Output パネルを開き、MCP または Copilot チャンネルでエラーを確認。
4. `npx @playwright/mcp --version` でパッケージが解決できるか確認。

### UI に表示されるバックエンド API エラー

- Spring Boot の起動を確認: `curl http://localhost:8080/api/policies`
- ターミナルで Java の起動エラーを確認 (ポート競合、Java バージョン不足など)。
- API に到達できない場合、フロントエンドは `"Failed to load data from backend"` を表示します。

### MCP でナビゲーションが即座に失敗する

- React アプリの起動を確認: `curl http://localhost:3000`
- MCP の `PLAYWRIGHT_BASE_URL` が `playwright.config.ts` の `baseURL` と一致していることを確認。

### 要素が見つからない

- まず `browser_snapshot` で現在のページ状態を確認。
- 正しいページ (Dashboard か Claims か) がロードされていることを確認。
- セクション 8 のセレクタ表にある正確な `data-testid` を使用。

### CLI テストが失敗するが MCP 操作は動く

- `npm run test:e2e:headed` でテスト実行中のブラウザを観察。
- タイミング問題を確認 — バックエンドのシードデータがまだ読み込まれていない可能性。
