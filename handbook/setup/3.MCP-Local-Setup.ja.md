# MCP セットアップ（ローカル限定）

> **このワークショップでは MCP をすべてローカルで動かします。**
> ⚠️ **社内ポリシーによりリモート MCP の利用は禁止です。** 外部にホストされた MCP
> サーバーへは接続しません。MCP は `.vscode/mcp.json` に定義し、**社内プライベートレジストリ**から
> `npx` でパッケージを取得して**自分の PC 上でローカル起動**します。
> **概念の解説:** [../guides/0.Concepts-Primer.ja.md](../guides/0.Concepts-Primer.ja.md) の §3。

---

## 0. 前提

- Node.js 18+（`npx` が使えること）
- GitHub PAT（`repo` / `read:org` / `workflow`）— [事前課題](1.Prerequisites.ja.md) で作成済み
- 社内ネットワーク/プロキシ越しに **社内プライベートレジストリ**へ到達できること

---

## 1. なぜローカル限定なのか（社内制約）

| 観点 | 本ワークショップの方針 |
| --- | --- |
| リモート MCP（http/SSE で外部ホスト）| **使用禁止**（社内ポリシー）|
| ローカル MCP（`stdio` / `npx` でローカル起動）| **これのみ使用** |
| パッケージ取得元 | **社内プライベートレジストリ**（npm public ではなく社内）|
| 設定スコープ | **ワークスペース**（`<repo>/.vscode/mcp.json`）|

> 全員が同じ構成・同じソースで揃うよう、**リポジトリ直下の `.vscode/mcp.json`（ワークスペース
> スコープ）** を使います。ユーザー/マシンスコープへの登録は行いません。

---

## 2. npm レジストリを社内プライベートレジストリに向ける

`npx` が社内プライベートレジストリからパッケージを取得するよう、レジストリを設定します
（具体的な URL・認証は当日、運営から共有されます）。

```bash
# 例: リポジトリ直下に .npmrc を置く、または以下を実行
npm config set registry <PRIVATE_NPM_REGISTRY_URL>
# 認証が必要な場合は運営の手順に従って .npmrc にトークンを設定
```

> ❗ public な npm レジストリに出てしまわないよう、`.npmrc` の設定を必ず先に行ってください。

---

## 3. `.vscode/mcp.json` を用意する

✅ **本リポジトリには `.vscode/mcp.json` が同梱済み**です（clone / Codespace 作成時点で配置済み）。
通常は**そのまま使えます**。PAT はファイルに保存されず、起動時に入力する方式です。

優先順位:
1. **運営提供の `mcp.json` がある場合はそれを最優先**（同梱版を置き換え）。
2. 無ければ**同梱版をそのまま使用**。
3. 何らかの理由で消えている場合のみ、テンプレートをコピー:

```bash
mkdir -p .vscode
cp handbook/setup/mcp.local.json .vscode/mcp.json
```

同梱／テンプレートの中身（[../../.vscode/mcp.json](../../.vscode/mcp.json) ／ [mcp.local.json](mcp.local.json)）:

```json
{
  "inputs": [
    {
      "type": "promptString",
      "id": "github-pat",
      "description": "GitHub Personal Access Token (scopes: repo, read:org, workflow)",
      "password": true
    }
  ],
  "servers": {
    "github": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": { "GITHUB_PERSONAL_ACCESS_TOKEN": "${input:github-pat}" }
    },
    "microsoft/playwright-mcp": {
      "type": "stdio",
      "command": "npx",
      "args": ["@playwright/mcp@latest"],
      "env": { "PLAYWRIGHT_BASE_URL": "http://localhost:3000" }
    }
  }
}
```

ポイント:
- すべて `"type": "stdio"`（= ローカル起動）。`http`/`sse` のリモート定義は使いません。
- PAT は `${input:github-pat}` で**起動のたびに入力**。ファイルにトークンを書き込みません。
- `.vscode/` は `.gitignore` 済み。PAT を含む設定が**誤ってコミットされない**ようになっています。

## 4. VS Code に認識させる

1. VS Code を開き直す（`Cmd+Shift+P` → **Developer: Reload Window**）
2. Copilot Chat を開く
3. 入力欄の **Tools（プラグ）アイコン**をクリック
4. `github` のツール群、`browser_*`（Playwright）が一覧に出れば成功
5. PAT の入力を求められたら、事前課題で作成した PAT を貼り付け

> ステータスバーの MCP インジケーターが緑になれば稼働中です。出ない場合は Output パネルで
> エラーを確認し、`.npmrc`（レジストリ/認証）と `.vscode/mcp.json` のパスを見直してください。

---

## 5. うまくいかないとき（当日の典型的な詰まり）

| 症状 | 確認ポイント |
| --- | --- |
| `npx` がパッケージを取得できない | `.npmrc` の社内レジストリ設定・認証トークン・プロキシ |
| ツールが Chat に出ない | Reload Window 後に再確認 / `.vscode/mcp.json` の位置（repo 直下）|
| GitHub ツールが 401/403 | PAT のスコープ（`repo`/`read:org`/`workflow`）・有効期限 |
| Playwright ツールが動かない | `npx playwright install chromium` を実行 |
| リモート MCP に繋ごうとして失敗 | **リモートは使いません。** ローカル（stdio）定義のみにする |

> 設定で時間が掛かっても、本編は MCP なしで進められます。Issue 作成は `gh`/Web UI、委任は
> github.com、UI 検証は `npm run test:e2e` で代替できます → **[4.MCP-Fallback.ja.md](4.MCP-Fallback.ja.md)**
> （[進行台本](../Instructor-Runbook.ja.md) のバックアップ早見表も参照）。
