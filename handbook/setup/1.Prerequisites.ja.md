## D. VS Code 拡張機能

Extensions Marketplace（`Cmd+Shift+X` / `Ctrl+Shift+X`）から以下をインストール:

**必須:**
- `GitHub Copilot Chat`
- `GitHub Pull Requests`
- `Extension Pack for Java`
- `Spring Boot Extension Pack`
- `ES7+ React/Redux/React-Native snippets`

> 📝 **拡張の最新事情（2025年）:**
> - **`GitHub Copilot`** は単体ではインストールしません。**`GitHub Copilot Chat` を入れると `GitHub Copilot` 拡張も自動的に一緒にインストール**されます（両者は統合済み）。
> - **`GitHub Pull Requests`** は旧称 **`GitHub Pull Requests and Issues`** から改名されたものです。名称に「Issues」は付きませんが、**Issue 機能は引き続き含まれます**。

> すべて入れたら VS Code を一度再起動して、拡張が有効化されることを確認してください。

## E. GitHub PAT の作成

MCP サーバーが GitHub にあなたとして認証するために使います。

1. **GitHub** → 右上のプロフィール → **Settings**
2. **Developer settings** → **Personal access tokens** → **Fine-grained tokens**（または **Tokens (classic)**）
3. **Generate new token**
4. わかりやすい名前（例 `workshop-mcp`）と **7 日以上**の有効期限を設定
5. スコープ: `repo` / `read:org` / `workflow` を付与
6. **Generate token** → 表示されたトークンを**その場でコピー**（再表示されません）
7. パスワードマネージャー等に安全に保管（MCP 設定時に使用）

> ⚠️ **Organization 配下リポジトリの場合:** Fine-grained PAT は **Resource owner = 対象 Org** を選び、
> Repository permissions に **Issues: Read and write** を付与してください。Org が「PAT 承認制」を
> 有効にしていると、**管理者がトークンを承認するまで使えません（リードタイムあり）**。承認は主催者が
> 前日までに段取りします（[0.Organizer-Preflight.ja.md](0.Organizer-Preflight.ja.md) §C）。承認が間に合わない場合は、
> ファシリテーターが Issue を**中央事前作成**するため、参加者は委任以降から参加できます。

```bash
# 動作確認（YOUR_PAT を置き換え）
curl -H "Authorization: token YOUR_PAT" https://api.github.com/user
# 期待値: あなたの GitHub ユーザー名を含む JSON
```

## F. リポジトリの clone とローカル起動確認

当日と同じ状態を再現できるよう、一度ローカルでアプリを起動しておきます。

```bash
# ワークショップ用リポジトリの URL は当日案内されます
git clone <your-metlife-repo-url>
cd <repo>/src
chmod +x run.sh
./run.sh
```

ブラウザで **http://localhost:3000** を開き、InsureWell のダッシュボードが表示されれば OK。

```bash
# 両サービスの起動確認
curl http://localhost:8080/api/policies   # ポリシーの JSON 配列が返る
curl http://localhost:3000                # HTML が返る
```
