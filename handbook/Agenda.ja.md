## 進行のポイント

- **前提は前日までに完了** — 当日の朝はインストールではなくスモークチェックから開始。
- **MCP はローカル限定**（リモート MCP 禁止）— `npx` は社内プライベートレジストリから取得。
- **概念プライマーを序盤に** — 日本語で BRD / MCP（原理・種類・スコープ）を先に共有してからハンズオン。
- **クラウドコーディングエージェントは昼直前に委任開始** — 昼休み中に非同期で走らせる。
- **スコープは 1 機能に固定**（例: 認証）— E2E フローを完走可能に。
- **各フェーズにバックアップ手順を用意** — 直線的な依存で前段が失敗しても進められるように（[進行台本](Instructor-Runbook.ja.md)）。
- **終盤にバッファ** — AI 生成コードのビルド不具合やトラブルを吸収。
- **完走が目的ではない** — 「新しい働き方の型」を体感してもらうことが目的（環境は後日も利用可）。
- **Azure DevOps CI は任意 / 宿題** — 本編は GitHub 中心に保つ。
